First Step: Why Component Development? (Part-1)

  March 14th, 2016

 Vision

During my development tenure I have worked on hundreds of web application. But I can add only few of them on my profile! Why? Because maximum web apps were developed for non-serious clients with no business vision, so after a year or so these web apps failed and clients didn’t even renewed the domains. And some were developed as junk (based on just do it, deliver rapidly with no security measures, no code quality, no QC, no application optimization etc). I believe in rapid development but there is no room for compromise on quality. Unfortunately local software market is all about quantity with no focus on quality and future perspectives. During my job I always preferred to adopt latest and stable technology according to project need and market demand. So I was not good fit for majority of local software houses, then I decide to quit my job and challenge myself by taking a bold decision and start working on my own ideas to develop a robust product with emphasis on quality, speed, security and scalability. Working on my own ideas gave me the freedom of choosing top of the line technologies that were best suited for my product.

 Market trends and component development

The market of component development for web or mobile application will not end. In this market there is huge space for who which comes with strong idea. Strong idea always becomes true using proper development flow(idea, design, develop, test, documentation, publish) and marketing strategy. There is no shortcut to achieve goals in this market. A proper team(with their perfect skills) with passion could make reputation in component development market. The start is full of obsticals, but the future is so bright. Following are some market trends or some reasons why components have extensive demand in the market

 Wordpress

WordPress is widely used CMS for small to medium businesses. WordPress is open source with great community. WordPress comes with numbers of themes and extensions. In cost effective components market WordPress is leading CMS in the market. In market there are many commercial theming frameworks and extensions available. The theming framework help user to change styles of their website. These theming framework comes with huge number of build-in templates and multipurpose themes. The extensions components provides the solution of specific business element.

 Components are cost effective

Buying component instead of the developing whole functionality from scratch, that's cost effective. The user has a choice to consider alternate component which fulfill his need. Most component's comes with 1 year software license. During this period they offer different kind of service like support, tickets etc.

 Standardized Code

Commercial component's comes with well written optimized code. It has extensive support(live, tickets, forums etc) with complete documentation. Component development companies follow market trends to develop their products. They used latest or stable technology to build components. These things made component first choice to replace development from scratch method.

 Migrations

Migration is very important part of software development. Migration has many types like data migration, platform migration, server migration, version migration etc. There are two type of version release occurred for components. First when base technology comes with newer version(like drupal7 to drupal8). Second type of release may contain new features or some bug fixes. Component development may provides the migration patch(small extension), which help user to migrate easily. These all facilities depends on software component license types.

 UI Components

UI components are the most demanding product of the component market, because in web/mobile application every things is done with it. We could be easily divided any web/mobile application as components, For example menus/navigation, slider, image gallery, news etc.

 What customer wants?

The component user want a useful control which comes with more features and flexibility but no complex configuration. They want buy component, single click install and start working. For this reason, mostly UI components are popular in the market as compare to backed components which comes with complex configuration.

 The enterprise components market

This market belongs to software development agencies or developers only. These components used to build larger application. These component's come with complex configuration. This kind of components come with multi-usage software license. The company which buy component, it could use to build any number of applications for selling purpose. This kind of component are costly.

 My Approach

My approach is similar like others, but I add some exceptional points to deliver unique products to user. Following are given below.

  •  Using KISS(principle) “Keep it simple, stupid” , I divided larger component into small extensions.
  •  Fewer dependencies among extensions of component.
  •  Provide maximum way to draw components (Provides multiple options for extensive resource consuming tasks)
  •  Build component for multiple platforms, like “SmartListing” does.
  •  Provides helpers(small util libraries) for different technologies
  •  Gave more features and build-in controls or theme templates for component
  •  Leave room for extendable and help to integrate custom or 3rd party controls
  •  New features suggested by users. Voting for current features
  •  Provides customization services on demand
