 Code

“SmartListings” code is written in well formatted and documented form. Every Control has many options for different kind of functionality. HandlebarJS is used as JavaScript rendering engine. I used JQuery for cross browser issues. All components are written in simple and concise form. “SmartListings” controls comes with multiple themes & control variations. Every control have one base type. User can extend that base type to create new control variation. “SmartListing” comes with multiple control variations (e.g, a CheckboxGroup have three variations according to functionality, i.e, CheckboxGroup,ScrollCheckboxGroup,PopupCheckboxGroup). “SmartListing” component have extensive quantities of controls themes and layout options.  Flexibility

The finest feature of “SmartListings” is flexibility. These components designed for developer, designer of any kind of user. The developer easily extend the current functionality of any control. The designer can easily change layout templates, or create new themes for “SmartListings” controls. These controls follow the simple flow of working.

 Extendable

“SmartListings” provide the following features to extend functionality.

  • 1) User have flexibility to create new component and registered with “SmartListings” components.
  • 2) User can extend any control to create new control type
  • 3) User could add any 3rd party control plugin with “SmartListings” (e.g, Select2 JQuery plugin)

 Performance Strategies

 Multiple HTML Rendering

The “SmartListings” provides the diverse approach to create UI controls for web page. The control could be created by server-side rendering engine(e.g, Twig, Blade etc) or JavaScript rendering engines (e.g, Handlebar, ejs etc). It depends on the required functionality. In simple functionality, user could use any approach to create UI controls. But in complex approach, mostly both approaches works together. The dynamic control created by JavaSscript rendering engine, because it need to dynamically filled with data(i.e, control options or values etc).

 Use Minimal Options

These components use multiple strategies for control creation (i.e, a Checkbox could be created using server side language or client side language like JavaScript or HTML). “SmartListings” provides the library files according to user needs. Its total depends on condition, what's users requirements. This saves the control's creation life cycle. Dynamic Control creation always sucks the browser memory. Minimal option provides the facility to overcome browser memory leak issues. In minimal option, the control created by server-side language (i.e, controls in Twig) and events implemented by JavaScript. This approach mostly used for traditional HTTP web application where the page refresh when any event fired.

 JavaScript Events

“SmartListings” components have a centralized event register, which keep track of every event. This event register properly disposed dynamic created events. This event register provides help for debugging memory related issues.

 Why I write code in VanillaJS instead of JavaScript Frameworks?

Why I prefer to write “SmartListings” components in VanillaJS instead of well reputed JavaScript Frameworks like AngularJS or BackboneJS? I faced this question when I presented my product to anyone! So I want to explain why I choose VanillaJS over JavaScript Frameworks

 Point#1

I started to write my component(SmartListings) in BackboneJS. Because, BackboneJS is less size and have very strong features. In simple control rendering Backbone Views were working fine. When I started to render nested control using BackboneJS View, I faced performance issue. That process were consuming huge memory. My aim to build a high performance tool which could work on low memory was failing. I start digging BackboneJS source code. 

                _createElement: function(tagName) {
      	return document.createElement(tagName);
}

It is creating DOM element dynamically that is extensive process for memory.

 Point#2

Every JavaScript support VanillaJS to extends its functionality. It means “SmartListings” components could easily port to any modern JavaScript framework. This point provides the compatibility with any JavaScript framework.

 Point#3

Freedom of code, JavaScript provides the flexibility to write code with freedom(Build anything attitude).
